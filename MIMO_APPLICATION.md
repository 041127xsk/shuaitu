# MiMo Orbit Token 创造者激励计划 — 项目摘要

**总字数：约 1050 字**

## 一、背景与痛点

《率土之滨》拥有千万级玩家，但武将战法、阵容搭配、对战数据等核心信息至今没有官方 API 接口。玩家只能手动翻阅攻略、截图识别战报、口口相传经验，效率极低且容易出错。

现有解决方案要么依赖人工整理（更新慢、覆盖不全），要么采用简单爬虫（只能抓取原始文本，无法理解语义）。更关键的是，游戏内战报以图片形式存在，传统 OCR 对艺术字体、特效遮挡的识别准确率不足，导致大量数据无法被有效利用。

## 二、解决方案与技术架构

我们构建了一套 **AI 驱动的全链路游戏数据采集与分析系统**，核心采用 **多智能体协作架构**，将复杂任务分解为多个协同的子任务：

**数据采集层**：通过 ADB 连接安卓模拟器，自动截图、滑动翻页、识别战报。采用 **分层 OCR 引擎**——基础层使用 RapidOCR 快速处理结构化数据（~50ms/张），增强层在置信度不足时自动调用 **通义千问 VL 视觉模型** 进行二次识别，确保复杂场景的准确率。

**语义理解层**：将攻略文章交给大语言模型进行 **结构化抽取**。每个武将的战法描述（数百字散文）需被解析为 skill_type、trigger_rate、targets、effects 等十多个字段。这要求模型具备 **长上下文理解能力**（单篇 3000-5000 字）和 **精准信息抽取能力**。

**智能分析层**：基于结构化数据构建阵容评分和模拟对战系统，需要模型理解武将克制关系、技能联动、阵营加成等复杂逻辑，进行 **多步推理**。

我们在开发中深度使用了 **OpenCode、Cursor、自定义 Agent** 等工具，形成了 **人机协作的开发闭环**——人类定义需求，AI 执行实现，人类验证结果。

## 三、Token 消耗预估

Token 消耗主要来自三个场景：

**战法结构化抽取**：每个武将战法约 500-1000 字，645 个武将全量抽取约 **50-80 万 Token**。考虑到迭代优化和错误重试，月均消耗约 **200-300 万 Token**。

**战报 OCR 增强**：每次识别需将图片发送给视觉模型，单次约 2000-3000 Token。按日采集 1000 条战报计算，日消耗约 **200-300 万 Token**，月消耗约 **6000-9000 万 Token**。

**智能分析与推理**：阵容评分和模拟对战需要模型理解复杂武将关系，单次推理约 5000-10000 Token。随用户量增长，日消耗可达 **500-1000 万 Token**。

**综合预估**：未来 6 个月，项目平均日消耗 **1000-5000 万 Token**，峰值可达 **1 亿 Token/天**。主要由战报采集的高频调用和智能分析的深度推理驱动。

## 四、创新性与未来潜力

本项目核心创新在于 **将大语言模型深度融入游戏数据处理全链路**，而非仅作为问答工具。我们验证了：

**多模态融合**：OCR + 视觉模型 + 文本模型协同，解决单一技术无法处理的复杂场景。

**分层置信度评估**：数值置信度（40%）+ 结构化校验（60%）混合评分，智能决定是否调用视觉模型。

**增量式数据采集**：断点续跑、去重机制、状态管理，确保大规模采集的可靠性。

小米 MiMo 模型特别适合我们的场景：**强大的中文理解能力**可精准解析游戏攻略中的专业术语；**高效的推理性能**可支撑高频战报分析；**开放的 API 生态**可无缝集成到多智能体架构中。

项目成熟后，我们将为千万率土玩家提供 **实时、准确、智能** 的配将建议和战报分析，彻底改变策略游戏的数据消费方式。这不仅是技术项目，更是 **AI 赋能游戏社区的典范**。
