### 4. 分层 OCR 战报采集系统（核心成果）

#### 架构
```
RapidOCR (50ms) → 混合评分 → < 0.75 → Qwen VL API (2s)
                    ↓
         数值置信度 (40%) + 结构化校验 (60%)
```

#### 新建文件
| 文件 | 功能 |
|------|------|
| `src/stzb_ocr.py` | 分层 OCR 引擎主入口 |
| `src/ocr_paddle.py` | RapidOCR 封装（PaddleOCR 3.x 兼容问题，用 RapidOCR 代替） |
| `src/qwen_vl_ocr.py` | Qwen VL API 封装（通义千问视觉模型） |
| `src/emulator.py` | ADB 模拟器控制（截图/点击/滑动） |
| `src/report_parser.py` | 战报解析器（支持同盟列表多条解析） |
| `scripts/collect_battle_reports.py` | CLI 批量采集脚本 |
| `scripts/server.py` | 本地 API 服务（FastAPI） |

#### 混合置信度评估
```
结构化校验 (60%):
  ├─ 武将名命中率 (35%) - 识别的武将名是否在已知库中
  ├─ 战报完整性 (35%) - 有胜负/玩家名/时间戳
  └─ 阵容合理性 (30%) - 有≥2个武将

数值置信度 (40%):
  └─ RapidOCR 返回的平均置信度
```

#### 滚动采集参数（已测试）
- 屏幕：1080x1920（MuMu 模拟器）
- 滑动：x=540, y=850→400, 距离=450px, 速度=400ms
- 去重：相同玩家 + 相同队伍不重复入库

#### 识别效果
- RapidOCR：72 文本块，置信度 95%+
- Qwen VL：准确识别严颜（不是颜良）、朱儁（不是朱偽）
- 自动分层：置信度高时用 RapidOCR，低时自动调 VL
